Regular expressions

Regular expressions, (sometimes referred to as Regex or even RegExp) are a great way of expressing complex rules in order to find information more quickly and with a higher degree of selectivity.

Because this is a broad and complicated topic, I will not be elaborating in any detail on how to use regular expressions as it has been already better explained online. See: https://www.regular-expressions.info/, in particularly: https://www.regular-expressions.info/quickstart.html.

Note: when browsing external sources for information regarding regular expressions make sure that you only use constructs that are supported by the .Net Framework regex engine. See .Net regex engine below for more details.

.Net regex engine

Regexes come in a multitude of different flavours with varying rules all dictated by which Regular Expression engine is being used. ReSearcher uses the regex engine provided by the .Net Framework. More information on the specifics of this engine can be found at: https://docs.microsoft.com/en-us/dotnet/standard/base-types/regular-expressions
